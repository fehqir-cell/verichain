# Security Policy

## Supported Versions

VerifChain is currently in active development. Security updates and patches will be provided for the latest deployment.

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| Older   | :x:                |

## Reporting a Vulnerability

We take the security of VerifChain very seriously, especially concerning the integrity of the fact-checking protocols and TON wallet integrations.

If you discover a security vulnerability, please do **NOT** open a public issue. Instead, please report it privately to ensure the safety of our users.

**How to report:**
1. Send an email to **verifchain@gmail.com**.
2. Include a detailed description of the vulnerability, the steps to reproduce it, and any potential impact (especially if it involves the smart contracts or TON Connect API).

**What to expect:**
- We will acknowledge receipt of your vulnerability report within 48 hours.
- We will send you regular updates about our progress in resolving the issue.
- Once the vulnerability is patched, you will be credited in our release notes for responsibly disclosing the bug.
